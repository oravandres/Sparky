# Media generation (image + video)

Sparky exposes async image and video generation through the gateway. Jobs
are persisted to a single-node, file-backed ledger and executed by
[`services/sparky-worker`](../services/sparky-worker/) (which polls the
ledger and drives ComfyUI). The source of truth for request/response
shapes is [`config/api-contract.yaml`](../config/api-contract.yaml); PLAN
authority: §5.5 (media), §5.7 (job control), §16 (media runtime), §18
(job execution model).

## Endpoints exposed by the gateway

Every route below requires `Authorization: Bearer <SPARKY_API_KEY>`
(PLAN §5, §10).

| Method | Path                          | Returns | PLAN §§      |
| ------ | ----------------------------- | ------- | ------------ |
| POST   | `/v1/media/image/jobs`        | `202`   | §5.5, §16    |
| POST   | `/v1/media/video/jobs`        | `202`   | §5.5, §16    |
| GET    | `/v1/jobs/{job_id}`           | `200`   | §5.7, §18    |
| POST   | `/v1/jobs/{job_id}/cancel`    | `200`   | §5.7, §18    |

`/v1/jobs/*` are shared between media (image, video) and audio (tts,
asr — landing in a follow-up PR). The persisted record carries a `type`
field so consumers branch on the job class without polling per-family
endpoints.

## Image jobs (`POST /v1/media/image/jobs`)

```json
{
  "model": "flux2-dev | flux2-klein | qwen-image | hunyuanimage-3-instruct",
  "prompt": "string (1..32000 chars)",
  "negative_prompt": "string (optional, ≤ 32000 chars)",
  "width": 1024,
  "height": 1024,
  "steps": 30,
  "seed": 0,
  "metadata": { }
}
```

Validation that fails with HTTP **422** before enqueue:

- `width` / `height` must be a multiple of **64** and within `[256, 4096]`.
- `steps` must be in `[1, 150]`.
- `seed` must be in `[0, 2^32 - 1]`.
- `model` must reference an *active* registry entry whose `family` is
  `image` and whose `runtime` is `comfyui`.
- Excluded models (`flux.1`, `flux.1-dev`, `sdxl`, `stable-diffusion-3.x`,
  …) are rejected explicitly — Sparky never silently substitutes an older
  model (PLAN §2.2).

A successful submission returns `JobAccepted`:

```json
{
  "job_id": "uuid",
  "type": "image",
  "status": "queued"
}
```

## Video jobs (`POST /v1/media/video/jobs`)

```json
{
  "model": "ltx-2 | wan-2.2 | hunyuanvideo-1.5",
  "prompt": "string (1..32000 chars)",
  "duration_seconds": 5,
  "fps": 24,
  "width": 1280,
  "height": 720,
  "max_frames": 120,
  "max_pixel_frames": 110592000,
  "seed": 0,
  "metadata": { }
}
```

Phase 1 envelope (relax via deliberate `config/api-contract.yaml` PRs only):

- `duration_seconds` ∈ `[1, 12]`, `fps` ∈ `[1, 24]`.
- `width` ≤ 1280, `height` ≤ 720, both `multipleOf` **16**.
- `max_frames` must equal `ceil(duration_seconds × fps)` and be ≤ 288.
- `max_pixel_frames` must equal `max_frames × width × height` and be
  ≤ 265 420 800 (288 frames at 1280×720).

Mismatched envelope values are an HTTP **422** before enqueue. This
keeps a single oversized job from monopolizing the single-slot media
tier (PLAN §4.3 Tier B).

## Job control (`/v1/jobs/{job_id}`)

`GET /v1/jobs/{job_id}` returns the persisted record:

```json
{
  "job_id": "uuid",
  "type": "image | video | tts | asr",
  "model": "string",
  "status": "queued | running | completed | failed | cancelled",
  "created_at": "iso8601",
  "started_at": "iso8601 (optional)",
  "completed_at": "iso8601 (optional)",
  "output_uri": "string (optional)",
  "error": "string (optional)"
}
```

The `request` payload Sparky received is intentionally **not** echoed
in this response — it remains on disk for the worker but is not part of
the public contract.

`POST /v1/jobs/{job_id}/cancel` transitions a non-terminal job to
`cancelled` and returns the updated record. State machine:

| Current `status` | After `cancel`         |
| ---------------- | ---------------------- |
| `queued`         | `cancelled` (200)      |
| `running`        | `cancelled` (200)      |
| `cancelled`      | `cancelled` (200, idempotent) |
| `completed`      | unchanged, **409**     |
| `failed`         | unchanged, **409**     |

`409 Conflict` carries `error.code = "job_terminal"` so callers can
distinguish "already done" from "not found" (`404 not_found`).

## File-backed job ledger (PLAN §18)

The gateway writes one JSON file per job under `JOBS_DIR` (default
`/var/lib/sparky/jobs`, see `config/sparky.env.example`). Writes use
`tempfile + os.replace` so a reader never observes a partially-written
record. Single-node only; multi-node scheduling stays with MiMi
(PLAN §1.2). The worker (PLAN §7.2) consumes the same directory.

`job_id` is a UUIDv4 generated by the gateway. The job-control routes
validate the path component before opening files — values like
`../etc/passwd` short-circuit to **404** instead of touching disk.

### Provisioning + boot resilience

The deployed gateway runs read-only with a non-root user (PLAN §10,
`docker-compose.gateway.yml`), so the host **must** bind-mount a
writable jobs directory. The Ansible base role
([`roles/base/tasks/main.yml`](../roles/base/tasks/main.yml)) creates
`/var/lib/sparky/jobs` as `sparky:sparky` mode `0750`; the compose
service mounts it back in:

```yaml
volumes:
  - /var/lib/sparky/jobs:/var/lib/sparky/jobs
```

To keep `/health` green when an operator forgets the mount, `JobStore`
**defers** directory creation: boot stores the path only, the first
submission lazily `mkdir(parents=True, exist_ok=True)`s it, and
`/ready` reports `jobs_dir.status="not_ready"` with the configured
path so MiMi monitoring catches the misconfiguration before any
caller does. Submissions on an unwritable jobs dir return
**`503 jobs_dir_unavailable`** with a stable envelope (no FS path
leakage in the public message — the path lives in `request_id`-tagged
gateway logs).

exercises submit → query → cancel against a running gateway. ComfyUI is
**not** required for the smoke test — the gateway only enqueues; actual
generation is the worker's job.
